## 技术栈
### 后端
- **框架**：Spring Boot 4
- **java**：java 25
- **AI集成**：Spring AI
- **数据库**：Spring Data JPA、PostgreSQL(version17 with pgvector)
- **设计模式**：DDD领域驱动设计、restful api

## 开发规范
- 本项目采用最新最前沿的技术栈，所以在构建项目时可以使用各种高级特性或语法（如java的虚拟线程、switch模式匹配、record类等），如果有不确定的，请调用context7工具查阅相关文档。
- 在引入依赖时，请你考虑当前已有的依赖（pom.xml）能否满足需求；如需引入依赖，请使用较新的依赖版本；项目的规范是尽量地精简依赖。
- 在每次开发之前，请你先了解当前的文件目录结构。
- 每个目录下适当阅读部分代码了解已有逻辑和代码风格。
- 在开发过程中遇到的一些报错可能只是误报（与环境有关），请你自行判断，小错误可以忽略。
- 如无说明，你不需要编写单元测试。
- 每次代码编写完成后，无需启动项目，也无需进行集成测试或测试接口。
